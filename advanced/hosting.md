# Hosting

Hosting is optional. Do not publish because the files exist. Publish only when there is a clear audience and review path.

## Use when

- A guide, site, or teaching resource is ready for readers.
- The user has approved public release.
- The repo has a repeatable build/check command.

## Do not use when

- Draft content is still unstable.
- The user has not approved publishing.
- Credentials, analytics, domains, or paid services would be needed without approval.

## Common paths

```text
GitHub Pages: simple static docs or guide site.
Netlify/Vercel: static sites with previews.
Manual export: one-off HTML/PDF handoff.
```

## Approval needed

```text
publishing or deploying
custom domains
analytics
paid services
secrets or tokens
changing GitHub workflow files
```

## Definition of Done

```text
audience is clear
build/check command passes
public/private boundary is reviewed
rollback path exists
release notes or handoff exists
```
