Agent Workflow Starter

A small repo for using Claude Code or another file-editing agent to produce checked writing, research notes, teaching material, docs, scripts, and small project artifacts.

Get startedOpen GitHub repo

Start

Start with the repo

Clone or fork the repo, open it locally, then give Claude Code one bounded task.

git clone https://github.com/logohere/agent-workflow-starter.git
cd agent-workflow-starter

1. State the goal

Name artifact, audience, scope, non-goals, and done condition.

2. Inspect files

Read only the files needed for the task.

3. Write the plan

List files touched, steps, checks, and follow-ups not included.

4. Challenge the plan

Ask what is missing, risky, too broad, or needs approval.

5. Make the change

Use small edits. Keep unrelated ideas out.

6. Hand off

Record what changed, what passed, and what remains.

Core rule

Do not let the agent own the goal

AI can draft, search, compare, organize, script, and check. The human keeps intent, judgment, standards, approval, and final review.

Copy this

First prompt for Claude Code

You are working in my project repo.
Inspect README.md, CLAUDE.md, agent-ops.md, index.html, human-guide.html, and bootstrap/bootstrap.md.
Do not edit yet.
State the current purpose, likely next task, checks available, and anything that needs approval.

Make work reviewable

Use a plan, small changes, checks, and a handoff. Do not ask for broad improvement.

Bounded task

Give the agent a target

Create or update [artifact] for [audience].
Use [files] as source material.
Keep the scope to [scope].
Do not change [non-goals].
Before editing, write an implementation plan.
After editing, run checks and summarize the diff.

Plan review

Question the plan before execution

What is missing?
What could break?
What assumptions might be wrong?
What is too broad?
What depends on something else?
What should be deferred?
What needs approval?
What is the smallest useful version?

Context

Spend tokens on evidence

Instead of pasting every file.Search first. Read only relevant sections.
Instead of arguing in a long chat.Write a handoff and restart from files.
Instead of asking for a full rewrite.Ask for a diff and the reason for each change.
Instead of repeating lookup.Use local SQLite search or a small script.

Restart

Start fresh when the chat is steering the work wrong

Wrong assumption repeats.Stop. Write the correct current state into a handoff.
Agent edits before inspecting.Stop. Restart from the project, issue, PR, and relevant files.
Task changed direction.Record the old decision as abandoned and state the new goal.
Several plans are mixed.Choose one plan and defer the rest.

Search

Choose the search type before adding tools

Exact lookupYou know the filename, heading, command, ID, or phrase.
Keyword / FTS5You know the words likely used in the file.
Fuzzy searchYou expect typos, partial words, spelling drift, or near matches.
Semantic searchYou know the idea but not the words; usually needs embeddings and vector search.
Hybrid searchYou need keyword precision plus semantic recall.
